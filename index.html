Перелік основних команд Git

  • git config --global user.name blabla--встановлює ім'я користувача
  • git config --global user.email blabal@bla--встановлює електронну пошту користувача
  • git init--ініціалізація репозиторія
  • git status--показує стан файлів в робочій директорії та індексі
  • git add--додає вміст робочої директорії в індекс для подальшого комміта
  • git commit--бере всі дані, додані за допомогою попередньої команди і зберігає їх відбиток у внутрішній базі даних, а потім переміщує вказівник поточної гілки на цей відбиток
  • git log--перегляд історії коммітів
  • git checkout--для перемикання гілок і завантаження їх в робочу директорію
  • git branch--менеджер гілок( створює нові, перечисляє, видаляє, перейменовує їх)
  • git push--передає зміни у файлі з локального до віддаленого репозиторію
  • git pull--бере зміни з вказаного віддаленого репозиторія і зливає їх з поточною гілкою
  • git merge--для злиття однієї або кількох гілок в поточну
  • git clone--вибір існуючого репозиторію і створення його копії
  • git revert--для безпечної відміни змін, дія цієї команди направлена у майбутнє
  • git reset--використовують для видалення змін
  • git rebase--ще один спосіб перенести зміни з однієї гілки в іншу. Стискає всі зміни в один патч, потім інтегрує патч в потрібну гілку
  • git cherry-pick--бере зміни, що вносить один коміт і поворює їх у вигляді нового коміта в поточній гілці
  • git stash--для тимчасового зберігання всіх незакомічених змін для очистки робочої директорії без необхідності комітити незавершену роботу в нову гілку
.gitignore файл--це файл, явним чином помічений для Git як файл, який необхідно ігнорувати
